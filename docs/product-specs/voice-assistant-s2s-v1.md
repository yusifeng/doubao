# Voice Assistant S2S V1 Product Spec

## 1. 文档目标

定义 V1 的产品规格（what），用于指导后续技术方案设计（how）与执行计划拆解。

## 2. 业务目标

- 构建一个可稳定语音聊天的助手（语音输入 + 语音输出）。
- 对齐参考产品的核心体验：对话列表、会话页、语音页三段式结构。
- 在不引入复杂能力的前提下，优先打通主链路并可稳定演示。

## 3. 范围与约束

### 3.1 In Scope（V1 要做）

- 会话列表页：展示历史会话，支持进入会话与新建会话。
- 会话详情页：展示消息流、文本输入、语音入口。
- 语音会话页：显示“听/想/说”状态，支持停止与退出。
- 实时语音对话主链路：用户说话 -> 识别 -> 助手回复文本 -> 助手播报语音。
- 消息基础操作：复制文本、重试回复（最小实现）。
- 样式 Tokenizer Design：建立统一设计令牌（颜色/间距/圆角/字体层级），页面样式仅消费令牌。

### 3.2 Out of Scope（V1 不做）

- 多角色市场与复杂人设编辑。
- 联网搜索、外部 RAG、插件系统。
- 多端同步与账号体系。
- 高级音色管理、唱歌模式等扩展能力。

## 4. 已确认的产品决策

基于本次需求对齐，V1 固定如下：

- 语音交互模式：**免长按**（非 push-to-talk）。
- 平台优先级：**Android 优先**。
- 助手形态：**通用助手（非强角色扮演）**。

## 5. 核心用户故事

- 作为用户，我希望打开 App 后可以直接开始语音对话，不必长按录音。
- 作为用户，我希望在对话过程中能看到识别/思考/播报状态，知道系统正在做什么。
- 作为用户，我希望保留历史会话，方便回看之前内容并继续聊。
- 作为用户，我希望在网络波动或失败时看到可理解的提示并快速重试。

## 6. 页面与信息架构

## 6.1 对话列表页

- 列表项字段：标题、摘要、更新时间。
- 交互：点击进入会话；支持新建会话。

## 6.2 会话详情页

- 消息流：区分用户消息与助手消息。
- 输入区：文本输入框、语音入口按钮。
- 助手消息操作：复制、重试（V1 最小集合）。

## 6.3 语音会话页

- 顶部：场景/设置入口（可占位）。
- 中部：助手头像与状态提示。
- 底部：麦克风、其他入口、结束/挂断按钮。

## 7. 状态模型（产品层）

统一状态机：

- `idle`：空闲可发起对话
- `listening`：正在听用户输入
- `thinking`：正在生成回复
- `speaking`：正在语音播报
- `error`：当前轮失败，用户可重试

状态切换要求：

- 界面状态与实际链路状态一致，避免“看起来在听，实际已断流”。
- 任一失败场景都要可恢复到 `idle`。

## 8. 语音事件流（产品语义）

对接字节 S2S Realtime WebSocket 文本能力（参考文档）：
- [端到端实时语音大模型API接入文档](https://www.volcengine.com/docs/6561/1594356?lang=zh)

V1 默认事件流程（稳定优先）：

1. 本机语音识别开始（ASR）
2. 用户结束输入后生成最终识别文本
3. 建立/复用 S2S 会话并发送文本 query
4. 接收助手文本回复
5. 本机 TTS 播报
6. 回合结束并回到空闲态

兼容策略：

- 实时音频流链路保留为 feature flag（`EXPO_PUBLIC_VOICE_PIPELINE_MODE=realtime_audio`），默认关闭。

## 9. 非功能需求（V1）

- 首包反馈可感知（有明确“正在听/思考”提示）。
- 断线、超时、鉴权失败时有明确错误文案。
- 会话操作可回退：退出语音页后不丢当前会话内容。
- 日志最小可观测：关键阶段有可排查记录（连接、识别、回复、播报、失败）。
- UI 令牌一致性：核心页面不允许散落硬编码视觉值（颜色/字号/间距）。

## 10. 验收标准

- Android 真机/模拟器可完成至少 5 轮连续语音对话。
- 用户无需长按即可开始一轮语音输入。
- 能看到并区分 `listening / thinking / speaking / error` 四类状态。
- 历史会话可见，点击后能回看消息流。
- 失败后支持重试并可继续对话。
- 核心页面样式由统一 token 驱动，至少覆盖颜色与排版层级。

## 11. 待技术方案决策（下一阶段）

- 音频采集与播放 SDK 选型。
- 本地存储方案（会话/消息持久化）。
- 会话状态管理方式。
- S2S 接入方式（直连或服务端中转）。
- Token 承载介质（NativeWind theme 扩展或独立 token 模块）。
